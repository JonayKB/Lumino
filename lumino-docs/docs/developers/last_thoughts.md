## Reflexiones finales

!!! quote "Aprendizajes clave"
    *“Un buen diseño inicial reduce drásticamente los problemas futuros.”*

!!! success "Lecciones aprendidas"
    - La documentación es tan importante como el código  
    - La reutilización de componentes reduce errores  
    - Automatizar procesos ahorra tiempo y esfuerzo  

??? tip "Recomendaciones para futuros proyectos"
    - Definir requisitos con claridad  
    - Mantener una estructura modular  
    - Documentar desde el inicio  

!!! failure "Errores evitados"
    Gracias a la planificación y buenas prácticas, se evitaron:

    - Reescrituras innecesarias  
    - Dependencias mal gestionadas  
    - Problemas graves en producción  

---

## Visión a largo plazo

!!! info "Futuro del proyecto"
    Lumino aspira a convertirse en una plataforma sólida, escalable y fácil de mantener, capaz de adaptarse a nuevas necesidades educativas.

!!! success "Cierre final"
    Este proyecto demuestra que una combinación de **buena arquitectura**, **documentación clara** y **mantenimiento continuo** es clave para el éxito de cualquier software.
